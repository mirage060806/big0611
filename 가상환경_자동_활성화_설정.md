# 가상환경 자동 활성화 설정 과정

## 문제
- `.venv` 가상환경 폴더가 존재하지만 터미널을 열 때마다 `(.venv)`가 표시되지 않음
- 터미널을 닫으면 가상환경 활성화가 해제됨
- 매번 `source .venv/Scripts/activate`를 입력해야 함

## 해결 과정

### 1. 가상환경 확인
- `.venv` 폴더가 프로젝트 루트에 존재하는지 확인
- 가상환경 구조 확인 (Scripts, Lib, Include 폴더 존재)

### 2. VS Code 설정 시도 (Devin 데스크톱에서 작동하지 않음)
- `.vscode/settings.json`에 Python 인터프리터 경로 설정 시도
- `python.defaultInterpreterPath` 및 `python.terminal.activateEnvironment` 설정
- Devin 데스크톱은 VS Code 기반이지만 터미널 설정이 다름

### 3. 터미널 설정 파일 생성
- Git Bash는 사용자 홈 디렉토리의 `.bashrc`를 자동으로 로드
- 사용자 홈 디렉토리 확인: `C:/Users/pc06-00`

### 4. 최종 설정
- 사용자 홈 디레토리에 `.bashrc` 파일 생성
- 파일 경로: `C:/Users/pc06-00/.bashrc`
- 파일 내용:
  ```bash
  # 가상환경 자동 활성화
  source e:/wi/git/big0611/.venv/Scripts/activate
  ```

### 5. VS Code 터미널 설정
- `.vscode/settings.json`에 Git Bash 기본 프로필 설정
- `--login -i` 옵션으로 로그인 쉘 실행 (`.bashrc` 자동 로드)

## 결과
- 터미널을 열 때마다 자동으로 가상환경이 활성화됨
- 프롬프트에 `(.venv)`가 자동으로 표시됨
- 매번 수동으로 활성화할 필요 없음

## 참고
- 설정 파일 위치: `C:/Users/pc06-00/.bashrc`
- 가상환경 경로: `e:/wi/git/big0611/.venv/Scripts/activate`
